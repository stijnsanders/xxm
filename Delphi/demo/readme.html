xxm Demo

Thank you for checking out xxm.
This package has a minimal set of executables and ready-built xxm projects to demonstrate the basic features of xxm.

To get started, unpack this package in a new folder and start xxmHttp.exe (it starts silently, it's normal that you don't get feedback), then open a web-browser and navigate to http://localhost/

In case you're already running a web-service locally, start xxmHttp.exe Port=8080, then navigate a browser to http://localhost:8080/, optionally with another available port number than 8080.

To end xxmHttp.exe, use a task manager to kill the running process.

Auto-compile

To demonstrate the auto-compile feature, if you have Delphi installed locally (with a working DCC32.exe), start xxmHttpDev.exe, delete the .xxl-file of a selected demo project, then navigate to its according URL. xxmHttpDev will generate the required files, and call DC32.exe to (re-)build the .xxl-file and run it. If you feed like it, make some changes to the .xxm)files, and check your changes by refreshing the browser.

The xxm project registry, used by xxm handlers to locate the registered xxm projects. Also used by listprojects.xxl to list the available projects.
xxmProto.xxl
The very first xxm project, created manually. It served to build and verify the xxm handlers, and as a template for the code generator. It tests the basic form posting methods: GET, POST, and POST with multipart/form-data to allow file-uploading, and shows parse data of the request's headers.
