xxm Demo Projects - 11 Basic Authentication

This is a demo project showing how to protect one or more pages with a plain username and password.

Attention: This method doesn't apply a high level of security. Username and password combination is transmitted in the HTTP meta-data, and is vulnerable to man-in-the-middle attacks. Therefore it's advisable to force the client to use HTTPS, but even so if you want proper security you should look into proper authentication on the server level by dedicated security infrastructure.

Web.xxmp
The project file, contains an internal list of the files of the project, and other data about the project.
Use xxmProject.exe to edit and manage project settings.
Default.xxm
The website's default page. It is not protected.
Secret.xxm
This page is protected by asking for username and password. If none is available, the page responds with HTTP status code 401 and a WWW-Authenticate header causing the user agent to pop-up a dialog to enter username and password.
xxmp.pas
The project's main unit. When this file is missing form the project, xxm creates one for you.
Here, TXxmdemo.LoadPage contains a section of code commented out showing how to enable the password protection over the entire site.
